# Web Components + Modern CSS Conversion — Design

**Date:** 2026-06-10
**Branch:** `web-compontents`
**Status:** Approved

## Goal

Convert the site from a Python-generates-everything build (markdown → HTML via
BeautifulSoup templating, Sass → CSS) to:

- Hand-written HTML pages using vanilla web components for shared chrome.
- Plain modern CSS (no Sass anywhere).
- A small Python tool (uv-managed) whose only jobs are building the blog from
  markdown and running a local dev server.
- As little JavaScript as possible, with zero JS dependencies.

The visual design of the site does not change.

## 1. Site structure

The repo root is the served site (GitHub Pages, `CNAME` preserved), as today.

**Hand-written HTML** (authored once from the current generated output, cleaned
up to use the new components):

- `/index.html` (home)
- `/about/index.html`
- `/projects/index.html`
- `/projects/vim-config-with-YouCompleteMe/index.html`

**Generated HTML** (built by the Python tool):

- `/blog/index.html` — post listing
- `/blog/<slug>/index.html` — one per post

Fonts, icons, and images stay where they are. Page features carried over from
the current template: `darkreader-lock` meta, CSP meta tag, font preloads,
Clicky analytics snippet in the footer.

## 2. Web components

One vanilla ES module at `/components.js`, zero dependencies, loaded by every
page. Light DOM only — no shadow DOM — so global CSS styles everything.

- `<site-header>` — renders the `h1` ("Andy Stanish" home link) and the main
  nav (about / blog / projects), marking the current section by matching
  `location.pathname`.
- `<site-footer>` — renders the social-media links and the Clicky badge.
- `<sub-nav>` — wraps and styles the right-side sub navigation. Its links are
  plain light-DOM children written into the page itself (hand-written on the
  projects pages, generated by Python on blog pages). The component does not
  fetch or compute the link list, so post data has a single source (the
  builder) and the links render even without JS.

Trade-off accepted: with JS disabled, the header and footer do not render;
page content and sub-nav links still do.

## 3. CSS

No Sass. Same visual design, translated to modern CSS:

- `/styles.css` — global styles, translated from the numbered Sass files
  (`00-global`, `01-header`, `02-footer`, `03-light`) and the partials
  (`_breakpoints`, `_code`, `_sub-nav`). Uses custom properties, native CSS
  nesting, and `prefers-color-scheme` for the light theme, as today.
- Per-page CSS files stay in place (`home.css`, `about/about.css`,
  `blog/blog.css`, `projects/projects.css`, per-post CSS) and are rewritten as
  plain modern CSS.
- Pygments syntax-highlighting CSS is generated at build time by the Python
  tool — no JS highlighting in the browser.
- Old root-level compiled artifacts (`_code.css`, `_sub-nav.css`) are folded
  into `styles.css` and removed.

## 4. Python blog builder

`src/` becomes a uv project:

- `src/pyproject.toml` — runtime deps: `markdown`, `Pygments`; dev deps:
  `pytest`, `pytest-cov`. No BeautifulSoup, no livereload.
- `src/posts/<slug>.md` — post sources, with markdown meta (title, date). The
  existing post (`tips-for-waking-up-far-far-too-early`) migrates here.
- `src/post_template.html` — page shell with the web components; templating is
  plain string substitution (no HTML-parsing library).
- `uv run build` — converts every post to `/blog/<slug>/index.html`, generates
  `/blog/index.html` listing posts (with dates, newest first), and writes the
  Pygments CSS.
- `uv run serve` — stdlib `http.server` on port 5500 serving the repo root,
  with an mtime-polling watcher that rebuilds the blog when a post or template
  changes. Browser refresh is manual (no livereload).
- A missing or malformed post meta block fails the build loudly with the
  offending file named.
- Developed with TDD; tests in `src/test/`.

## Testing

- Builder: pytest unit tests (TDD) for markdown conversion, meta parsing,
  index generation, and path handling; side effects (file writes) localized
  per the existing code-style rules.
- Site: manual verification in the dev server that every page renders with
  header/footer/sub-nav and matches the current visual design in dark and
  light schemes.

## Out of scope

- Any visual redesign.
- RSS/Atom feeds, tags, pagination (YAGNI — one post exists today).
- Deleting-a-page automation from the old TODO.
